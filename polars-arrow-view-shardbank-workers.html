Arrow-View Shard Bank Workers (Polars precursor)

This page is the next experiment inspired by the Arrow-view shim idea. It builds a partitioned shared WebAssembly.Memory shard bank, exposes it through an Arrow-style manifest of live buffers, runs distributed worker aggregates directly over those live views, and then performs a mutation proof in a fresh worker set.

Important: this is a precursor to direct Polars live-view integration, not that final step yet. The point here is to prove the live shared-bank view, partition model, and mutation semantics without going through Arrow IPC or another serializer.

Ready. Build the shared bank, run the distributed live-view aggregate, then verify the mutation delta.
Live-view setup

Each partition stores its column buffers fully inside single shards. The manifest is Arrow-like: each column chunk points at values in a specific shard + offset + length.
