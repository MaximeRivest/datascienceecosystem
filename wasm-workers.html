MVT 1.5: Shared Wasm Memory → 4 Web Workers

This prototype replaces the raw SharedArrayBuffer with a shared WebAssembly.Memory. The same memory is imported into 4 workers, and a Rust-compiled Wasm function sums assigned ranges.

Checking browser + Wasm shared-memory requirements...
Single shared Wasm memory above 4 GiB is usually unavailable in current Chromium unless memory64 support is present.
Test setup

Use this to validate zero-copy shared data access from Wasm workers. For an 8 GiB goal, first try 8192 MiB here. If allocation fails, that means a single shared Wasm memory is not available at that size in this browser/runtime. In that case, use multiple shards.
